# Demo Request Inbox

参考サイト、スクショ、動画をここにまとめて置いてください。
まずは `request.md` に追記し、素材があれば `assets/` に入れるだけで十分です。

## 一番簡単な共有方法

1. `demo-requests/assets/` に参考動画、スクショ、GIFを入れる。
2. `demo-requests/request.md` に以下の2つだけ書く。
   - 参考URL
   - demo化したい箇所
3. 複数demoをまとめて書いて問題ありません。上から順番に実装されます。
4. 素材ファイル名が分かる場合だけ、該当行に書く。

## 完了判定とWIPチップ

実装は一度で完成扱いにしません。
依頼者が「OK」「これで完了」など明確に承認するまでは、そのdemoは作業中として扱います。

- OK前: ページやカードに `WIP` チップを付ける。
- OK後: `WIP` チップを外し、必要なら実装メモや発信用文面を整える。
- 判断に迷う場合: OKが出ていないものは `WIP` のままにする。

この運用にすると、まとめて依頼したdemoのうち、どれがまだ調整中か一覧で分かります。

## 参考サイトと素材の使い分け

実装時は、参考動画やスクショだけを見て作りません。
必ず参考URLの実サイトを開き、実際のDOM、ホバー、スクロール、レスポンシブ挙動を確認してから実装します。

- 参考URL: 仕様の主な参照元。動き、余白、状態変化、レスポンシブ挙動を確認する。
- 動画/GIF/スクショ: どの部分をdemo化したいかを特定するための補助資料として使う。
- 実サイトと素材に差がある場合: 実サイトを優先し、差分が大きい場合は実装前または実装メモで明記する。
- 実サイトが見られない場合: その理由を伝え、素材ベースで進めるか確認する。

## 最小フォーマット

```md
### Demo名 または 仮名
- 参考URL:
- 対象箇所:
- 素材: assets/example.mp4
- Status: WIP
```

## もう少し詳しく伝えたい場合

```md
### Demo名
- 参考URL:
- 対象箇所:
- 期待する動き:
- 再現度: 高め / 雰囲気だけ / このサイトに合わせてアレンジ
- 素材:
  - assets/example-scroll.mp4
  - assets/example-hover.png
- Status: WIP / OK
- 備考:
```

## 書き方の例

```md
### Agency Hero Split Text
- 参考URL: https://example.com
- 対象箇所: トップのファーストビュー。見出しが分割されて順番に出るところ。
- 素材: assets/agency-hero-scroll.mp4
- Status: WIP

### Product Card Hover
- 参考URL: https://example.com/work
- 対象箇所: 作品カード一覧のホバー。画像が少し拡大して、ラベルが追従するところ。
- 再現度: 雰囲気だけ。このサイトの一覧UIに合うように調整。
- Status: WIP
```

## 素材ファイル名のおすすめ

後で見返しやすいように、可能なら短い名前にしてください。

```text
agency-hero-scroll.mp4
product-card-hover.png
menu-transition.gif
loading-logo.mp4
```

以下は未記入で問題ありません。実装時にこちらで既存構成に合わせます。

- demoのslug
- 使用技術
- レイアウトの細かい寸法
- 実装ファイルの分け方
- サイト全体との見た目の統一
